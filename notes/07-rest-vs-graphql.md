# REST vs GraphQL

## REST
- Banyak endpoint, satu untuk setiap resource
- Response sering over-fetching/under-fetching data

## GraphQL
- Satu endpoint, client menentukan data yang diambil
- Lebih efisien untuk kebutuhan data yang kompleks
- Memiliki schema dan type system sendiri

Pilihan tergantung kebutuhan: REST simple untuk CRUD, GraphQL bagus untuk query kompleks.
