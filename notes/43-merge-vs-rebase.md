# Git Merge vs Rebase

## Merge
Menggabungkan dua branch dengan membuat merge commit baru, history tetap apa adanya.

## Rebase
Memindahkan commit dari satu branch ke ujung branch lain, menghasilkan history yang linear/bersih.

Rebase cocok untuk branch lokal sebelum di-push, hindari rebase pada branch yang sudah dibagikan ke orang lain.
